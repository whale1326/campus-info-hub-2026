## 一、项目概述

本次实训我选择开发"校园信息平台"（Campus Info Hub），一个集失物招领、二手交易、信息发布、评论留言于一体的全栈 Web 应用。项目采用前后端分离架构，前端使用 Next.js 15 + React 19 + TypeScript + Ant Design 5，后端使用 Python Flask + SQLite，通过 JWT 实现用户认证。

项目最终包含：

| 维度 | 数量 | 说明 |
|------|------|------|
| 前端路由 | 10 个 | 首页、登录、失物招领(列表+详情)、二手交易(列表+详情)、发布信息、个人中心、管理员(仪表盘+用户+内容) |
| 后端 API | 22 个 | 认证(5)、帖子(6)、评论(3)、管理员(6)、统计(2) |
| 数据库表 | 3 张 | users、posts、comments |
| 单元测试 | 12 个 | 覆盖认证、CRUD、统计等核心功能 |
| Git 提交 | 17 次 | 分功能模块多次提交 |

### 核心功能模块

1. **失物招领**：发布丢失/拾到物品信息，支持搜索筛选和状态管理
2. **二手交易**：发布闲置物品，展示价格，支持标记已交易
3. **信息发布**：发布校园通知、活动信息
4. **评论留言**：在帖子详情页发表评论，支持用户互动
5. **个人中心**：编辑个人资料、修改密码、管理我的发布
6. **管理员后台**：用户管理、内容审核、数据统计仪表盘

## 二、AI 工具运用

在本次实训中，我深度使用了 AI 编程工具（WorkBuddy/Claude）辅助项目开发，共发起 13 次 Prompt 交互，记录在 `prompt_log.md` 中。主要体现在以下几个方面：

1. **项目架构设计**：通过 Prompt 描述需求，让 AI 生成完整的项目目录结构、技术选型建议和文件组织方案。

2. **代码生成与迭代**：利用 AI 生成后端 Flask 应用（22 个 API、JWT 认证、权限控制）、前端 Next.js 页面（10 个路由、Ant Design 组件），以及数据库 Schema 和全套文档。

3. **问题排查与修复**：遇到 SQLite 写入失败（沙箱限制无法创建 journal 文件），通过 AI 分析根因并修复——将数据库路径迁移到用户主目录并设置 `PRAGMA journal_mode = MEMORY`。

4. **UI 美化优化**：通过 Prompt 让 AI 优化前端设计，实现了玻璃态导航栏、渐变色统计卡片、卡片阴影动画等现代 UI 效果。

5. **功能迭代开发**：在基础功能完成后，分阶段添加了管理员后台、个人中心、评论留言功能，每次都通过 Prompt 描述需求由 AI 生成代码。

6. **代码审查**：利用 AI 对项目进行全面的 Code Review，生成审查报告（`docs/code_review.md`），发现 18 个优化建议，涵盖安全性、代码质量、性能、架构和测试五个维度。

## 三、技术收获

1. **前后端分离架构**：理解了 Next.js 的 App Router 模式、Server/Client Component 的区别，以及通过 `rewrites` 配置实现 API 代理。

2. **JWT 认证与权限控制**：实现了完整的用户注册、登录、Token 签发验证流程，并设计了多层级权限控制（`login_required` 普通认证、`admin_required` 管理员认证），通过装饰器模式实现接口级权限管理。

3. **数据库设计**：设计了 3 张核心数据表，支持外键关联、级联删除、索引优化和 CHECK 约束，实现了分页查询、多条件筛选和全文搜索。

4. **工程化实践**：配置了 `.gitignore`、`requirements.txt`、CI/CD 等工程化文件，编写了 README.md、API 文档、Prompt 日志、代码审查报告和总结报告。

5. **单元测试**：使用 PyTest 编写 12 个测试用例，覆盖了健康检查、用户认证、信息 CRUD 和统计接口。

6. **Git 版本管理**：按照功能模块将开发过程拆分为 17 次提交，每次提交有清晰的 commit message，体现了良好的版本管理习惯。

## 四、代码审查发现

通过 AI 代码审查（详见 `docs/code_review.md`），发现以下主要问题：

- **严重 (1项)**：密码哈希使用 SHA-256，建议升级为 bcrypt
- **中等 (5项)**：硬编码默认密码、Secret Key 暴露、Debug 模式、SQL 字符串替换、测试覆盖不足
- **低 (12项)**：认证装饰器代码重复、分页参数异常处理、前端导航方式、缺少速率限制等

同时也确认了项目的优点：SQL 参数化查询防止注入、完善的权限控制、清晰的代码结构、完整的 TypeScript 类型定义。

## 五、不足与反思

1. **AI 依赖度较高**：项目中大部分代码由 AI 生成，虽然进行了审查和理解，但在某些实现细节上仍需深入学习。

2. **测试覆盖不完整**：12 个测试只覆盖了基础 CRUD，管理员 API、个人中心 API、评论 API 缺少测试用例。

3. **安全性待加强**：代码审查发现密码哈希算法不够安全，生产环境需升级为 bcrypt；登录接口缺少速率限制。

4. **前端缺少状态管理**：用户状态通过 localStorage 管理，未使用 React Context，导致状态变化时部分组件需要刷新才能更新。

## 六、总结

本次实训让我深刻体会到 AI 辅助编程的效率提升——从零到功能完整的全栈项目（22 个 API、10 个路由、3 大功能模块）在短时间内完成。通过分阶段的 Prompt 交互，我逐步构建了失物招领、二手交易、管理员后台、个人中心、评论留言等功能，并完成了代码审查和全套文档。

同时我也认识到，AI 生成的代码仍需人工审查——代码审查报告中发现的 18 个问题就是证明。理解每一行代码的含义、掌握安全编程实践、建立工程化思维，比单纯生成代码更重要。未来我将继续探索 AI 编程工具的更多可能性，同时夯实基础技术能力，做到"会用 AI，更懂原理"。
