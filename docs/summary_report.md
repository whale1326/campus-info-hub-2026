## 一、项目概述

本次实训我选择开发"校园信息平台"（Campus Info Hub），一个集失物招领、二手交易和信息发布于一体的全栈 Web 应用。项目采用前后端分离架构，前端使用 Next.js 15 + React 19 + TypeScript + Ant Design 5，后端使用 Python Flask + SQLite（开发环境）/ Supabase PostgreSQL（生产环境），并通过 Vercel 进行部署。

项目共包含 7 个前端路由页面（首页、登录、失物招领列表/详情、二手交易列表/详情、发布信息）和 9 个后端 API 接口（健康检查、用户注册/登录、信息 CRUD、统计数据），以及 12 个单元测试，全部通过。

## 二、AI 工具运用

在本次实训中，我深度使用了 AI 编程工具（WorkBuddy/Claude）辅助项目开发，主要体现在以下几个方面：

1. **项目架构设计**：通过 Prompt 描述需求，让 AI 生成完整的项目目录结构、技术选型建议和文件组织方案，大大缩短了前期规划时间。

2. **代码生成与迭代**：利用 AI 一次性生成后端 Flask 应用（含 JWT 认证、分页查询、错误处理）、前端 Next.js 页面（含 Ant Design 组件集成、API 调用封装），以及数据库 Schema 和全套文档。生成的代码结构清晰，经过少量调试即可运行。

3. **问题排查与修复**：在后端单元测试初次运行时，6 个测试因数据库路径硬编码而失败。通过 AI 快速定位问题并修复为动态配置读取方式，12 个测试全部通过。

4. **环境搭建**：借助 AI 完成了 Node.js、Python Flask、Vercel CLI、Supabase CLI 等开发工具的安装配置，以及 VS C++ 编译工具的安装指引。

所有 AI 交互的 Prompt 均记录在 `prompt_log.md` 中，体现了从需求描述到代码生成的完整过程。

## 三、技术收获

1. **前后端分离架构**：通过实际项目理解了 Next.js 的 App Router 模式、Server Component 与 Client Component 的区别，以及如何通过 `rewrites` 配置实现开发环境的 API 代理。

2. **JWT 认证机制**：在后端实现了完整的用户注册、登录、Token 签发与验证流程，加深了对 JWT 原理和安全实践的理解。

3. **数据库设计**：设计了包含用户表、信息表的两张核心数据表，支持分类筛选（失物招领/二手交易）、分页查询和统计聚合，同时编写了 Supabase 的 RLS（行级安全）策略。

4. **工程化实践**：配置了 `.gitignore`、`requirements.txt`、`pytest.ini` 等工程化文件，编写了 README.md 和 API 文档，培养了良好的工程规范意识。

5. **单元测试**：使用 PyTest 编写了 12 个测试用例，覆盖了健康检查、用户认证、信息 CRUD 和统计接口，保证了后端代码的可靠性。

## 四、不足与反思

1. **AI 依赖度较高**：项目中大部分代码由 AI 生成，虽然我进行了审查和理解，但在某些实现细节（如 Next.js 的 SSR/SSG 机制）上仍需深入学习。

2. **前端测试缺失**：由于时间限制，前端部分未编写单元测试，后续可补充 Jest + React Testing Library 的测试用例。

3. **部署尚未完成**：受限于本地环境配置，项目尚未完成线上部署，后续需将前端部署至 Vercel、后端部署至 Render/Railway。

4. **功能可扩展性**：当前功能较为基础，后续可增加图片上传、实时通知、搜索优化等功能提升用户体验。

## 五、总结

本次实训让我深刻体会到 AI 辅助编程的效率提升——从零到可运行的全栈项目仅用了数小时，这在传统开发模式下是难以想象的。但同时也认识到，AI 生成的代码仍需人工审查和调试，理解每一行代码的含义比单纯生成代码更重要。未来我将继续探索 AI 编程工具的更多可能性，同时夯实基础技术能力，做到"会用 AI，更懂原理"。
